作成日: 2026-06-21

## 1. 目的と背景

`bqn-ledger` を設定駆動化するにあたり、封筒予算（`role=budget`）を使った予算配賦を行わず、単純に「資産・負債・収入・支出」の実績と予定のみを管理したいというライフスタイル（例: 普遍的な一般家計簿）をサポートします。

現在、コードベースは封筒が存在することを暗慢の前提としている部分があり、予算アカウントが一切定義されていない場合にエラー（境界外アクセスやゼロ除算など）になる懸念があります。これらを排除し、安全に予算機能を「オプショナル」として扱えるように設計を整理します。

---

## 2. 各レイヤーにおける設計方針

### 2.1 コア層 (`src/core/build_cube.bqn`)
* **現状**: 予算の delta 射影（`ProjectBudgetConsumption`, `ProjectBudgetAlloc`）は、アカウントマスタに予算マップが存在する場合のみアクティブになります。
* **挙動**: 予算アカウント（`role=budget`）が1つも存在しない場合、`budget_map` はすべて空（または ¯1）になり、`budget` レイヤー（layer 2）および `budget_alloc_sum` レイヤー（layer 3）への射影 delta は常に 0 になります。
* **判断**: コア層は、コードを一切変更せずとも、予算なしの状態で「常に 0 が書き込まれた Daily Cube」を問題なく構築できます。

### 2.2 コンテキスト解決 (`src/core/context.bqn`)
* **課題**: `config.tsv` では `BUDGET_ID_SPENT` や `BUDGET_PREFIX` などの項目が必須項目として検証されます。
* **解決策**: 
  - 封筒予算なしで運用する場合でも、設定エラーを避けるために `config.tsv` にはプレースホルダーとしてのダミー値（例: `BUDGET_PREFIX = budget`、`BUDGET_ID_SPENT = budget:spent`）を記述した状態とします。
  - アカウントマスタ（`accounts.tsv`）側には対応するアカウントを一切定義しない（または `role=` を割り当てない）ことで、実質的に無効化します。

### 2.3 ビュー層 (`src/views/envelope_view.bqn`)
* **課題**: アカウントマスタから `role=budget` を持つアカウントインデックス（`env_idxs`）を抽出する際、配列の長さが 0 になります。この状態で平均日額や健康ステータスなどの計算に進むと、エラーを誘発します。
* **解決策**: 
  - `envelope_view.bqn` 内で、対象となる予算アカウントの個数をチェックします。
  - 個数が 0 の場合は、計算処理をバイパスし、空のリストや安全なダミーレコードを返すようにガードを挟みます。

### 2.5 レポート・表示層 (`src/reports/main.bqn`)
* **解決策**: 
  - メインレポートの構築入口、あるいは `report_sections.bqn` にて、「予算アカウントが存在するかどうか」を事前に判定します。
  - 予算アカウントが存在しない場合、`envelopes` セクションの構築および表示処理を完全にスキップ（非表示）にします。
  - これにより、封筒を使わない家計簿のレポート出力には、純粋に資産・収支・予定情報だけが表示されるようになります。

---

## 3. 具体的な変更予定

### 3.1 `src/views/envelope_view.bqn`
予算アカウント（`env_idxs`）が空（`0 = ≠ env_idxs`）の場合、計算をスキップして空の namespace リストを返すガードを追加します。

```bqn
# 擬似コード
(0 ≠ ≠ env_idxs) ◶ ⟨
  { 𝕊 ⋄ 空のビューデータを返す },
  { 𝕊 ⋄ 従来の集計・計算処理を実行 }
⟩ @
```

### 3.2 `src/reports/main.bqn` または `src/reports/report_sections.bqn`
セクションリストから `envelopes` を除外する判定を追加します。
`accounts.tsv` から `role=budget` に該当するアカウントがあるか、または context に定義された `budget_prefix` で始まるアクティブなアカウントが存在するかで判定します。

## 4. テスト検証計画 (Fixture A: generalization-calendar)

封筒予算しない場合の挙動を検証するため、以下の Fixture A を作成します。

* **フォルダ名**: `fixtures/generalization-calendar`
* **構成ファイル**:
  - `accounts.tsv` (予算アカウントなし、`role=asset|liability|income|expense` のみ)
  - `cycle.tsv` (`mode=calendarMonth` を指定)
  - `config.tsv` (コンテキスト用のダミー設定値を保持)
  - `journal.tsv` (給与、家賃、食費、クレカ決済などの普遍的な取引データ)
  - `plan.tsv` (固定費予定、通信費予定など)
  - ※ `budget_alloc.tsv` は配置しない

### 期待される結果
- `./checks/check.sh` がエラーなしで通過すること。
- `main.bqn` をこの Fixture に対して実行した際、`[envelopes]` セクションがクラッシュせず、綺麗に非表示になること。
- `export-envelope-summary.bqn` などの予算系エクスポートスクリプトも、空データに対して正常終了（ヘッダーのみ出力など）すること。
