# A Technique for Opening from One Person to the World

This repository is not only a ledger report engine.

It is a practical art for opening from one person's life to the world.

## Note

`technique` is used here in the older sense of *art*, *craft*, and *skill*: a lived way of handling something, not merely an efficient procedure.

This project begins from one person's concrete life: money, pain, limits, plans, debt, spending, saving, reports, and the need not to break the source data.

But it does not end inside one person.

Through AI conversation, BQN, TSV, bookkeeping, arrays, reports, Go editor commands, and everyday use, the work changes how the person meets the outside world: friends, helpers, institutions, books, software, time, color, attention, and relation.

The aim is not to cure a person by naming a condition.

The aim is to build a small usable form around a life, so that the life can be seen, handled, protected, and opened.

## Working sentence

```text
A Technique for Opening from One Person to the World
```

In Japanese:

```text
一人から世界へ開くための技法
```

This is a small phrase for the deeper orientation of the repository.

- `data/*.tsv` is the readable ground.
- BQN is the array waterway that turns records into reports.
- The Go editor is a small gate for safe life operations.
- UI tools are outer clothing, not the source of truth.
- AI is a companion for design, inspection, explanation, and careful making, not an owner of the source data.

The user may be one person, but the practice is not closed.

It opens from one life toward the world.
