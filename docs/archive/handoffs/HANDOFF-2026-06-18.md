# Handoff: as_of report semantics hardening

Date: 2026-06-18  
Status: active handoff / can be archived after follow-up work is done  
Commit: `1824a12 Harden as-of report semantics`

## Why this exists

moko wants the current report outputs to remain stable while cleaning up time-axis behavior step by step.

The important operational decision made during this work:

```text
journal.tsv = actual records only
plan.tsv    = future / declared plans
future-dated journal rows = input mistakes
```

This document summarizes what changed, what is now fixed by fixtures, and what remains.

## What changed in commit `1824a12`

### 1. Section-by-section `as_of` audit

Added:

- `docs/AS_OF_SECTION_AUDIT.md`

It records how each human report section applies `as_of`, what was corrected, and what remains suspicious.

### 2. Characterization fixture for future journal rows

Added:

- `fixtures/future-journal-as-of/`

Purpose:

- Keeps current behavior visible when `journal.tsv` contains rows after `--as-of`.
- This is a characterization fixture, not approval of future journal rows.
- Sec8 now warns about these rows because they are considered input mistakes.

### 3. `last_journal_date` fixed

Changed:

- `src/views/plan_view.bqn`

Old behavior:

- `last_journal_date` effectively became `as_of`, because `as_of` was appended and then the last element was taken.

New behavior:

- If journal rows exist: `last_journal_date = max(journal date)`.
- If journal is empty: fallback to `as_of`.
- `journal_lag_days` now reflects the gap between `as_of` and last recorded journal coordinate.

Plan-open filtering was kept separate so normal future plan behavior does not depend on `last_journal_date`.

### 4. Envelope `avg/day` month-boundary bug fixed

Changed:

- `src/views/envelope_view.bqn`

Old behavior:

- elapsed days used numeric `YYYYMMDD` subtraction.

New behavior:

- elapsed days use ordinal date difference via `dt.DaysBetween`, inclusive of `as_of`.

Added:

- `fixtures/envelope-month-boundary/`

Regression case:

```text
cycle_start = 2026-01-31
as_of       = 2026-02-02
elapsed     = 3, not 72
```

### 5. YTD corrected to real year-to-date

Changed:

- `src/views/ytd_view.bqn`
- `src/reports/report_engine.bqn`

Old behavior:

- YTD used all loaded journal rows.

New behavior:

```text
as_of_year-01-01 <= journal.date <= as_of
```

### 6. Current cycle summary cut at `as_of`

Changed:

- `src/views/cycle_view.bqn`
- `src/reports/report_engine.bqn`

Old behavior:

- Cycle summary used all journal rows inside the resolved cycle.

New behavior:

```text
cycle_start <= journal.date < cycle_end_exclusive
and journal.date <= as_of
```

Past-cycle compatibility fields still exist, but the public past-cycle feature remains disabled.

### 7. Future journal warning added

Changed:

- `src/views/hygiene_view.bqn`
- `src/reports/report_engine.bqn`
- `src/reports/sections/sec_readiness.bqn`

Sec8 now displays:

```text
future journal rows after as_of : N
  - YYYY-MM-DD memo (amount)
```

This is currently a warning, not a hard error, because fixtures and historical `--as-of` checks intentionally include later source rows.

### 8. Plan lifecycle status table added

Changed:

- `src/views/plan_view.bqn`
- `src/reports/sections/sec_plan.bqn`
- `src/reports/report_engine.bqn`

`planned` section now has:

1. Future/open payments list
2. Plan Status table for all `plan.tsv` rows

Status values:

| status | meaning |
|---|---|
| `future_open` | plan date is after `as_of` and not completed |
| `due_open` | plan date equals `as_of` and not completed |
| `overdue_open` | plan date is before `as_of` and not completed |
| `completed` | journal has the same non-empty `plan_id` |

Completion rule:

```text
plan.tsv row with plan_id=X
journal.tsv row with plan_id=X
=> completed
```

The future payments list now excludes:

- completed plans
- overdue plans before `as_of`
- budget-to-budget plan moves

Overdue plans remain visible in the Plan Status table.

## Remaining work / next decisions

### 1. Residual semantics

Post-handoff follow-up completed after commit `274ced1`:

- `residual_view.bqn` receives `as_of` and `plan_status_table`.
- Actual expense rows are limited to the current cycle up to `as_of`.
- Plan expense rows are limited to lifecycle statuses:
  - `completed`
  - `due_open`
  - `overdue_open`
- `future_open` plans are excluded from residual until they become due/overdue/completed.
- `actual_only` rows with no plan remain visible as behavior-drift observations.

### 2. Future journal warning vs hard error

Post-handoff follow-up completed after commit `a45e60e`:

- `journal.date > as_of` remains a Sec8 warning for historical `--as-of` views.
- `journal.date > system_today` is now a lint/strict-check error.
- Future declarations belong in `plan.tsv`; `journal.tsv` remains actual records only.

Do not make `journal.date > as_of` a hard error blindly, because existing fixture / historical `--as-of` use cases intentionally view past observations over a source tree containing later rows.

### 3. Rename `last_journal_date`

The field now means max recorded journal coordinate, not row-tail date.

Possible future rename:

```text
last_journal_date -> last_recorded_on
```

If renamed, update:

- `src/reports/report_engine.bqn`
- `src/reports/sections/sec_outlook.bqn`
- exporters if any
- `docs/REPORT_FIELD_MAP.md`
- `docs/MAIN_SECTIONS.md`
- golden outputs

### 4. Go source TSV editor

Implementation is still not approved unless moko explicitly says to start.

Relevant design decisions already made:

- BQN remains canonical read/check/report engine.
- Go is source TSV editor / safe writer.
- `plan_id` is adopted as optional metadata.
- `plan_open` / `plan_all` distinction is adopted.
- `plan finish apply` remains disabled until recovery/idempotency design is complete.
