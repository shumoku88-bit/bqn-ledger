# 引き継ぎ (予算レイヤ導入 / bqn-ledger)

> NOTE: これは当時の引き継ぎメモ（アーカイブ）です。現行の設計/運用は `docs/ROADMAP.md` / `docs/MAINTENANCE.md` / `docs/ARCHITECTURE.md` を参照してください。

## 目的
- `journal.tsv` は **現実のお金の動き**だけを手入力（一次データ）
- `accounts.tsv` は **勘定科目 + ローカルルール（メタ情報）**の対応表
- 予算配賦は `budget_alloc.tsv` に手入力（予算の source of truth）
- 予算レイヤ（封筒/箱分け）の消費行（`budget:<箱> -> budget:spent`）は、`main.bqn` が `journal.tsv` と `accounts.tsv` の `budget=...` メタ情報から **オンザフライで自動導出**する
- `budget_journal.tsv` は（任意）互換用の書き出し先（手編集しない / 生成物 / `.gitignore`）

この分離により、手入力は `journal.tsv` 1行のまま、内部では2行（現実 + 予算）相当にしてレポートできる。

### ファイル構成（関係するもの）
- `journal.tsv`
  - 手入力。現実の複式簿記（実績）。
- `budget_alloc.tsv`
  - 手入力。予算の配賦（`budget:unassigned -> budget:*`、期首投入など）。
- `accounts.tsv`
  - 手入力。科目のメタ情報 `key=value` を持つ。
  - 例: `assets:smbc\ttype=liquid`
  - 例: `expenses:タバコ\tbudget=嗜好品`
- `budget_journal.tsv`
  - 自動生成（任意/互換用）。予算レイヤ（消費行）。通常は不要で `.gitignore`。
- `tools/gen-budget.bqn`
  - `journal.tsv` と `accounts.tsv` を読んで、`budget_journal.tsv` 相当の内容を **stdout に出力**する BQN ツール（`> budget_journal.tsv` で書き出し）。
- `core.bqn`
  - `LoadLines` が `#` と `\\` から始まるコメント行を無視する（生成物ヘッダ対策）。
- `main.bqn`
  - `journal.tsv` を読み込み、予算レイヤの消費行をオンザフライで導出して残高一覧を出力。
- `forecast.bqn`
  - 財務の見通し（資産サマリ / 収入日起点の見通し）を出力。

### 予算レイヤ（生成器）の仕様（現状）
- `journal.tsv` の各行について
  - `to` が `expenses:*` の場合のみ対象
  - `accounts.tsv` の `expenses:*` 行に `budget=<箱>` があれば
    - `budget:<箱> -> budget:spent <amount>` を `budget_journal.tsv` に生成する

## 使い方
```bash
bqn main.bqn
bqn forecast.bqn

# （任意）予算レイヤのTSVを書き出したい場合
bqn tools/gen-budget.bqn > budget_journal.tsv
```

## 現状の状態
- 予算箱の残高は `budget_alloc.tsv`（配賦）で決まる。
  - 例: `budget:opening -> budget:unassigned`（原資投入）
  - 例: `budget:unassigned -> budget:*`（箱への配賦）
- `main.bqn` の予算消費（自動導出）は `budget_alloc.tsv` の最初の日付以降の `journal.tsv` だけを対象にする（途中開始できる）。
