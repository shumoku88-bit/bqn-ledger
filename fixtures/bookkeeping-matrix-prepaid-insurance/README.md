# Bookkeeping Matrix Study: Prepaid Insurance and Expense Recognition

本フォルダは、簿記行列研究のStage 3として、1年分の保険料の前払いと1か月分の費用認識を検証するpublic synthetic fixtureです。production source truthではありません。

## 1. 研究対象

- 保険料の現金支払いと費用認識の時点の分離
- 未消費分を前払費用という資産として保持すること
- 1か月分だけを前払資産から保険費用へ振り替えること

## 2. 手計算仕訳

### 取引1: 1年分の保険料を前払い（2026-10-01）

1年分の保険料12,000 JPYを銀行口座から支払う。

```text
借方  assets:prepaid-insurance  12,000
貸方  assets:bank               12,000
```

### 取引2: 1か月分の保険費用を認識（2026-10-31）

1か月分1,000 JPYの保険サービスを受けたため、前払保険料から保険費用へ振り替える。

```text
借方  expenses:insurance         1,000
貸方  assets:prepaid-insurance   1,000
```

## 3. Posting IRの符号規約

借方方向を正、貸方方向を負とする。

- 前払保険料の増加: `+12000`
- 銀行資産の減少: `-12000`
- 保険費用の増加: `+1000`
- 前払保険料の減少: `-1000`

## 4. event行の意味

`expected-event-account-matrix.tsv` の各event行は合計0になる。これは各取引の借方合計と貸方合計が一致し、符号付きpostingがzero-sumであることを示す。

## 5. account列をscanする意味

各account列をevent順にBQNのscan `` +` `` で累積すると、各取引後のrunning balanceを導出できる。前払後は銀行`-12000`、前払保険料`12000`、保険費用`0`となり、1か月分の認識後は銀行`-12000`、前払保険料`11000`、保険費用`1000`となる。

## 6. 会計上の核心

前払時点では、将来の保険サービスに対して現金を支払っただけであり、まだサービスを受けていない部分は資産である。このため、取引1では`expenses:insurance`を増加させない。

費用認識時点では、新たな現金支払いではなく、受けたサービス1か月分を前払資産から費用へ振り替える。このため、取引2では`assets:bank`を動かさない。

最終的に、未消費の前払資産11,000 JPYと認識済み費用1,000 JPYが残る。`11000 + 1000 = 12000`は、当初支払額が未消費部分と消費済み部分へ配分されたことを表す。

## 7. 売掛金・未払金研究との違い

- 売掛金研究は、収益認識後に債権を現金回収した。
- 未払金研究は、費用認識後に債務を現金支払いした。
- 本研究は、現金を先に支払い、資産として保持した額の一部を後から費用認識する。

各fixtureとテストは相互依存させない。

## 8. 非対象

- 12か月すべての費用認識、自動償却、償却スケジュール
- periodic transaction、recurrence、月末締め処理エンジン
- 日割り、解約、返金、契約変更、消費税
- 複数契約、複数通貨、前払費用の包括的分類
- 未払費用との比較、前受収益、減価償却
- production loader / Posting IR adapter / Cube / TBDS / report接続
- editor、journal writer、source migration、private data
- 共通fixtureフレームワーク抽出や既存研究テストのリファクタリング
