# Receivable Bookkeeping Matrix Study (Stage 1)

## 1. 研究対象
- 掛け売上による売掛金の発生
- 売掛金の回収
- 売上認識と現金回収の分離

## 2. 手計算仕訳
通常の複式簿記表記による取引の仕訳は以下の通りです。

### 取引1: 掛け売上 (2026-09-01)
顧客への商品の引渡し、またはサービスの提供が完了し、対価3,000円を掛けとした。
```text
(借方) 売掛金  3,000  /  (貸方) 売上  3,000
```

### 取引2: 売掛金の回収 (2026-09-10)
売掛金3,000円が銀行口座へ入金された。
```text
(借方) 銀行    3,000  /  (貸方) 売掛金  3,000
```

## 3. 符号規約
本システム（および bqn-ledger）では以下の符号規約を使用します。
```text
借方方向の増加 = 正 (+)
貸方方向の増加 = 負 (-)
```
したがって、各項目の増減に対応する符号は以下のようになります。
- 資産の増加（売掛金発生時の売掛金、回収時の銀行）: 正 (+)
- 資産の減少（回収時の売掛金）: 負 (-)
- 収益の増加（売上発生時の売上）: 負 (-)

## 4. 行方向の意味
各イベント行（取引）の合計（`row_total`）は常に `0` になります。
これは、複式簿記における貸借平均の原理（借方合計 ＝ 貸方合計）および、取引ごとのゼロ和（取引内の符号付き posting の総和がゼロ）を表します。

## 5. 列方向の意味
各勘定科目（列）をイベント発生順に累積（累積和）することにより、各時点の勘定残高（Running Balances）を導出できます。

## 6. 会計上の核心
- **売上認識のタイミング**: このfixtureでは、商品の引渡しまたはサービスの提供が取引1の時点で完了したと仮定し、その時点で売上を認識します。支払条件が掛けであることは、現金回収の時期を後ろへずらしますが、売上認識を回収時まで遅らせません。
- **回収時の影響**: 回収（取引2）の時点では新たな売上は認識されません。資産の形態が「売掛金（債権）」から「銀行預金（金銭）」へと置き換わるのみです。

## 7. 非対象
この研究 fixture では、議論を極めて単純化するため以下の要素を意図的に除外しています。
- 消費税
- 売上原価・棚卸資産
- 貸倒損失・貸倒引当金
- 値引き・割引・割戻
- 振込手数料・回収手数料
- 外貨建取引
- 一部回収（部分回収）
- 複数顧客
- 買掛金（債務側）
