# Bookkeeping Matrix Study: Payable Recognition and Payment

本フォルダは、`shumoku88-bit/bqn-ledger` における簿記行列（Bookkeeping Matrix）研究の第2題として、買掛金の発生と支払いの会計イベントおよびその行列・残高変換を検証するための合成テスト用フィクスチャです。

## 1. 研究対象

* サービス費用の発生
* 買掛金の計上
* 買掛金の支払い
* 費用認識時点と現金支払い時点の分離

## 2. 手計算仕訳 (通常の借方・貸方表記)

### 取引1: サービス費用の発生と買掛金計上 (2026-09-15)
2,400円の保守サービスを受け、代金は後日支払う契約とした。
* 借方: 保守費用 2,400円
* 貸方: 買掛金 2,400円

### 取引2: 買掛金の支払い (2026-09-25)
買掛金 2,400円を銀行口座から支払った。
* 借方: 買掛金 2,400円
* 貸方: 銀行預金 2,400円

## 3. 符号規約 (Posting IR)

本リポジトリの Posting IR 符号規約に基づき、以下のように定義します。
* 借方方向 = 正 (`+`)
* 貸方方向 = 負 (`-`)

具体的には、本研究に用いる勘定科目の符号は以下の通りです。
* **費用の増加 (借方)**: 正 (`+`) -> `expenses:maintenance` の増加
* **負債の増加 (貸方)**: 負 (`-`) -> 買掛金の計上 (liabilities:accounts-payable は負の値になる)
* **負債の減少 (借方)**: 正 (`+`) -> 買掛金の支払い (liabilities:accounts-payable に正の値を加算して0にする)
* **銀行資産の減少 (貸方)**: 負 (`-`) -> `assets:bank` の減少

## 4. 行方向の意味 (貸借一致)

`expected-event-account-matrix.tsv` の行方向の合計（`row_total`）がすべて `0` になることは、すべての取引イベントにおいて貸借が一致（Double-entry balance invariant）していることを意味します。

## 5. 列方向の意味 (累積残高)

各取引イベント（行）の値を時系列順に列ごとに累積（BQN の `+`` scan）することで、各時点における各勘定科目の残高（Running Balances）が得られます。

## 6. 会計上の核心

* **費用の認識タイミング**: 費用は保守サービス（サービス提供）を受けた時点（2026-09-15）で認識されます。
* **支払いとの分離**: 買掛条件は現金支払いを後日に延期するだけであり、費用認識そのものを支払日まで延期しません。「買掛だから費用を認識する」のではなく、費用認識の正当な根拠は「保守サービスという便益を享受したこと」にあります。
* **支払い時の会計イベント**: 2026-09-25の支払い時には、新しい費用は一切発生しません。発生済みの負債（買掛金）が消滅し、銀行預金という資産が同額減少します。

## 7. 売掛金研究との鏡関係

* **売掛金研究 (Stage 1)**: 収益認識（売上）と代金回収の時点を分離して検証した。
* **買掛金研究 (Stage 2)**: 費用認識（保守費）と代金支払いの時点を分離して検証する。

※ ただし、テストやフィクスチャ同士は完全に独立しており、相互依存関係はありません。

## 8. 非対象 (本研究の範囲外)

本検証を単純かつ明瞭に保つため、以下の項目は意図的に対象外としています。
* 棚卸資産
* 仕入商品
* 売上原価
* 消費税
* 未払費用との厳密な区別
* 支払手数料
* 値引き
* 外貨
* 部分支払い
* 複数取引先
* 支払期日管理
* 資金繰り
* 売掛金
