# Bookkeeping Matrix Study: Unearned Maintenance Revenue

本フォルダは、簿記行列研究のStage 4として、年間保守サービス代金の前受けと1か月分の収益認識を検証するpublic synthetic fixtureです。production source truthではありません。

## 1. 研究対象

- 現金受領と収益認識の時点の分離
- 未提供サービス分を前受収益という負債として保持すること
- 1か月分だけを前受収益から保守収益へ振り替えること

## 2. 手計算仕訳

### 取引1: 年間保守サービス代を前受け（2026-11-01）

顧客から1年分の保守サービス代12,000 JPYを銀行口座で受け取る。

```text
借方  assets:bank                    12,000
貸方  liabilities:unearned-revenue  12,000
```

### 取引2: 1か月分の保守収益を認識（2026-11-30）

1か月分1,000 JPYのサービスを提供したため、前受収益から保守収益へ振り替える。

```text
借方  liabilities:unearned-revenue   1,000
貸方  income:maintenance              1,000
```

## 3. Posting IRの符号規約

借方方向を正、貸方方向を負とする。

- 銀行資産の増加: `+12000`
- 前受収益（負債）の増加: `-12000`
- 前受収益（負債）の減少: `+1000`
- 保守収益の増加: `-1000`

負債は貸方側で増えるため、前受収益の増加は負になる。収益も貸方側で増えるため、保守収益の増加は負になる。

## 4. event行の意味

`expected-event-account-matrix.tsv`の各event行は合計0になる。これは各取引の借方合計と貸方合計が一致し、符号付きpostingがzero-sumであることを示す。

## 5. account列をscanする意味

各account列をevent順にBQNのscan `` +` `` で累積すると、各取引後のrunning balanceを導出できる。前受金受領後は銀行`12000`、前受収益`-12000`、保守収益`0`となり、1か月分の認識後は銀行`12000`、前受収益`-11000`、保守収益`-1000`となる。

## 6. 会計上の核心

現金受領時点では、将来提供するサービスへの対価を受け取っただけである。未提供部分は収益ではなく顧客へサービスを提供する義務なので、前受収益という負債として保持する。このため取引1では`income:maintenance`を動かさない。

収益認識時点では、新たな現金受領ではなく、提供済みの1か月分を前受収益から収益へ振り替える。このため取引2では`assets:bank`を動かさない。

最終的に、未提供サービスに対応する前受収益11,000 JPYと認識済み保守収益1,000 JPYが残る。符号付きでは`-11000 + -1000 = -12000`、金額の大きさでは`11000 + 1000 = 12000`となり、受領額が未提供部分と提供済み部分へ配分されたことを表す。全勘定でも`12000 + -11000 + -1000 = 0`を保つ。

## 7. 4題の違い

```text
売掛金:
  収益を先に認識し、後から現金を受け取る

未払金:
  費用を先に認識し、後から現金を支払う

前払費用:
  現金を先に支払い、後から費用を認識する

前受収益:
  現金を先に受け取り、後から収益を認識する
```

Stage 4により、発生先行の売掛金・未払金と、現金先行の前払費用・前受収益という最初の4方向の有限研究セットが閉じる。各fixtureとテストは独立し、相互依存しない。

## 8. 非対象

- 12か月すべての収益認識、自動収益認識、償却・月次スケジュール生成
- periodic transaction、recurrence、月末締めエンジン、日割り計算
- 解約、返金、契約変更、値引き、消費税
- 複数顧客、複数契約、複数通貨、請求書、入金消込、サブスクリプション管理
- revenue recognition engine、前受金と前受収益の包括的分類、新しいaccount taxonomy
- production loader / Posting IR adapter / Cube / TBDS / report接続
- editor、journal writer、source migration、private data
- Journal Posting IR adapter parity Stage 2
- 共通fixture frameworkや既存簿記行列テストのリファクタリング
