# BQN-Ledger Project Instructions

## 生活のルール（AI用更新ガイドライン）

`plan.tsv` やその他の将来の予定を更新・変更する際は、以下の「生活のルール（型）」を必ず守ること。

*   **年金サイクル（隔月）**: 年金は**偶数月**の15日（休日の場合は直前の平日）に支給される。
*   **固定費の連動（隔月）**: 家賃、光熱費、借金返済、保険料は年金支給日に連動する（隔月発生）。
    *   `plan.tsv` において `anchor=income:年金` が設定されている行は、年金支給がない月（奇数月）にはコピーやロールオーバーを行わないこと。
*   **毎月発生する費用**: 通信費（wifi, povo）やサブスクリプション（gpt-plus, google-oneなど）は毎月発生する。これらは翌月へロールオーバーすること。
*   **予定金額の同期**: `plan.tsv` の将来の収入（年金、支援給付金など）の金額を更新する際は、必ず configured native Journal の最新の実績（Actual）を確認し、一致させること。
*   **セッション起動時の第1ターンの義務（重要）**: Checkpoint等からコンテキストが復元された場合であっても、セッション起動直後の最初のターンにおいては、一切の調査やコード編集に先立ち、必ず `AGENTS.md` の「初回到着時の手順」を実行すること。具体的には、`docs/AI_CODEMAP.md`、`TODO.md`、`docs/QUALITY_BAR.md` の 3 ファイルを順番に `view_file` で明示的にロードし、リポジトリの最新データ状態と品質基準を確実に脳内に同期してからでなければ、他のツールを使用してはならない。

これらのルールは、AIが意図せずサイクルを崩したり、二重計上したりするのを防ぐための重要な制約である。
